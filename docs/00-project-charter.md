# Project Charter

## Project Name

Task & Requests Control System

## Mission

Build an internal control system for the IT department that manages department requests, employee task assignment, task progress, comments, and audit history.

## Current Scope

The current system is based on the supplied schema and contains only:

- Roles
- Permissions
- Users
- Requests
- Tasks
- Task updates
- Comments
- Audit logs

## Cancelled For Now

The following earlier ideas are not part of the current system:

- Purchases / MR
- Products
- Assets
- Suppliers
- Attachments
- Notifications
- System settings

## Primary Users

- Super Admin
- IT Manager
- Supervisor
- Employee

## Success Standard

The IT manager should understand the real department status within 30 seconds from the dashboard.

The dashboard must show:

- Open requests
- Active tasks
- Overdue tasks
- Blocked tasks
- Employee workload
- Tasks waiting review
- Recent task updates
- Recent audit activity

## Out Of Scope For Stage 0

- Building the frontend
- Building the backend
- Connecting a production database
- Installing packages
- Creating production deployment

Stage 0 only defines the system clearly and prepares the project structure.
